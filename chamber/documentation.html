Configuration Documentation

Learn how to customize the Chamber of Commerce website using our configuration system

Chamber Configuration System

This website uses a global configuration object (chamberConfig) to customize various components without modifying the core code. This makes it easy to adapt the website for different chambers of commerce or to update content and appearance.

Header Configuration

The header component can be customized with the following options:

window.chamberConfig = {
    header: {
        // Logo settings
        logoSrc: 'images/logo.webp',
        logoAlt: 'Chamber of Commerce logo',
        
        // Header title text
        titleTop: 'Chamber of Commerce of',
        titleHighlight: 'Province of Cordoba',
        
        // Navigation menu items
        navItems: [
            { href: 'index.html', text: 'Home' },
            { href: 'directory.html', text: 'Directory' },
            { href: 'join.html', text: 'Join' },
            { href: 'discover.html', text: 'Discover' }
        ]
    }
}

Implementation Guide

To customize the website for a different chamber of commerce, follow these steps:

  1. Create a new JavaScript file (e.g., config.js) or add to an existing script file.
  2. Initialize the chamberConfig object with your desired settings.
  3. Include this file in all HTML pages before the header.js and footer.js scripts.

Example implementation:

<script>
    // Global configuration for chamber website
    window.chamberConfig = {
        header: {
            titleTop: 'Chamber of Commerce of',
            titleHighlight: 'Your City Name'
        },
        footer: {
            companyName: 'Chamber of Commerce of Your City Name',
            address: 'Your Address Here',
            email: 'your.email@example.com',
            phone: '(123) 456-7890'
        }
    };
</script>
<script src="scripts/header.js" defer></script>
<script src="scripts/footer.js" defer></script>

Extending the Configuration

The configuration system can be extended to support additional customization options. To add new configuration options:

  1. Add a new property to the chamberConfig object.
  2. Modify the relevant component script to check for and use the new configuration option.
  3. Provide fallback values for when the configuration option is not provided.

Example of extending a component:

// In your component script
window.chamberConfig = window.chamberConfig || {};
const config = window.chamberConfig.yourComponent || {};

// Access configuration with fallback values
const option1 = config.option1 || 'default value';
const option2 = config.option2 || 'default value';
